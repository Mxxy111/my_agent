# Markdown Agent Output: AI Tells

AI agent output (Claude Code, ChatGPT with markdown rendering, Cursor, Cline,
RAG report bots) has a distinctive "voice" that goes beyond what plain-text
AI tells cover. The agent is rewarded by RLHF for looking organized,
authoritative, and thorough — and it pushes those signals into format choices.

This file is the dedicated checklist for that document type. Use it when the
target document is:

- a markdown report produced by an AI agent
- a code review summary
- a RAG / search-engine answer
- a technical note generated by Claude / ChatGPT / Gemini in markdown
- a README or documentation file drafted by an AI

---

## Tell Group A: Header Inflation

### A1. `##` every 3 sentences

AI agents reach for a `##` whenever they sense a topic shift, even if the
shift is small. A 400-word paragraph might get 4 second-level headers.

Symptom:

```markdown
## 一、问题背景
xx 是一个常见问题。它影响了很多场景。

## 二、原因分析
原因主要有三个。

## 三、解决方案
我们可以采用 A 方法。
```

A human writing the same thing would just write one paragraph.

### A2. Nesting 3-4 levels deep

Real documents rarely need `####` and never need `######`. Agents do this
because RLHF rewards "structured thinking" appearance.

### A3. Title Case English / 顿号-style 中文

- English headers always written in Title Case ("How to Use the System") even
  in casual contexts.
- 中文标题前后必加冒号或顿号: "**核心要点：** xxx" / "**问题、原因与对策**".

### A4. Repeated header structure

Section after section uses identical sub-headers: `### 背景`, `### 现状`,
`### 建议`. Real human authors vary their structure as content demands.

## Tell Group B: Bold Strafing

### B1. Every noun is bold

Symptom: 8+ `**...**` per paragraph. Every key concept emphasized.

```markdown
**XGBoost** 是一种强大的 **梯度提升** 算法,在 **Kaggle 竞赛** 中被
广泛使用。它的 **核心优势** 包括 **高准确率**、**快速训练** 和 **正则化能力**。
```

This is bold-strafing, not emphasis. When everything is emphasized, nothing is.

### B2. Bold for terms that are not actually terms

`**首先**`, `**总的来说**`, `**值得注意的是**` — these are transitions, not
defined terms, but agents bold them anyway.

### B3. Bold inside bullet lists

Each bullet starts with `**Title:**` followed by content. This is the most
visible AI agent fingerprint in markdown documents.

```markdown
- **可扩展性:** 系统可以处理大规模数据。
- **可维护性:** 代码结构清晰。
- **可复用性:** 模块设计良好。
```

A human listmaker rarely formats this way. They either write a clean list or
write a paragraph.

## Tell Group C: Bullet Nesting Hell

### C1. One-line content split into 5 bullets

Symptom: ideas that fit in one sentence get atomized into 5 separate bullets.

AI version:

```markdown
- 性能优化的关键点:
  - **缓存策略:**
    - 使用 Redis
    - 设置合理的 TTL
  - **数据库优化:**
    - 添加索引
    - 避免 N+1 查询
```

Human version:

```markdown
性能优化主要看缓存和数据库两块: 缓存我用 Redis,TTL 设短一点;数据库就盯紧索引和 N+1。
```

### C2. Sub-bullets that just rephrase the parent

```markdown
- 用户认证
  - 实现用户认证功能
  - 确保认证安全
```

Three bullets, one idea, zero new information.

### C3. Bullets for sequential narrative

If your bullets read as a story (first this, then that, then this), it should
be prose. Bullets are for parallel items, not for narrative.

## Tell Group D: Emoji Decoration

### D1. Status emoji at section starts

`✅ 已完成 / ❌ 待修复 / 🔥 重点 / 💡 提示 / ⚡ 性能 / 🚀 部署`. Every header
gets one.

### D2. Emoji as bullet markers

```markdown
- ✅ 性能优化
- 🔧 工具改进
- 📊 数据分析
```

### D3. Emoji inside body text

"我们的目标是提升性能 🚀 同时降低成本 💰". Real writers rarely sprinkle emoji
through prose.

### D4. Emoji in `##` headers

`## 🎯 核心目标` / `## 📋 总结`. The combination of emoji + Chinese header is
especially diagnostic for Chinese AI agent output.

## Tell Group E: Table Strafing

### E1. Tables for two-thing comparisons

Any time the agent compares A and B, it produces a table, even when one
sentence ("A is faster but B uses less memory") would be clearer.

### E2. Tables with one or two real data rows

The header row exists. The first data row exists. The next 3 rows say
"待定 / TBD / N/A". Real authors don't ship empty tables.

### E3. "Pros and Cons" tables

```markdown
| 优点 | 缺点 |
|---|---|
| 速度快 | 内存占用大 |
| 易用 | 学习曲线陡峭 |
```

This is one of the most overused agent shapes.

### E4. Tables for things that are not actually tabular

Sequential steps in a table. Definitions in a table. Long prose in a table cell.

## Tell Group F: Task List Hallucination

### F1. Unrelated `- [ ]` checklists

The agent output contains task lists pretending to track project progress,
even when no actual task tracking is happening.

```markdown
- [x] 分析问题
- [x] 设计方案
- [ ] 实现代码
- [ ] 测试验证
```

For a one-shot answer, this is theater, not utility.

### F2. Self-generated TODO lists

Agents like to end reports with "## TODO" or "## Next Steps" containing
generic items that the agent itself invented.

## Tell Group G: Code Block Abuse

### G1. Code block for filenames

```
`config.json`
```

is fine. But:

````
```
config.json
```
````

is the agent's habit. Inline code is nearly always better for short identifiers.

### G2. Code block for short prose

```
```
这是一段说明文字。
```
```

Code fences should be reserved for actual code, commands, or fixed-width data
(like file paths or output snippets). Don't use them for prose emphasis.

### G3. Triple-fenced "example" boxes

The agent loves to wrap any example in fenced code blocks regardless of
content type. A real example often reads better as inline quoted text or
indented prose.

## Tell Group H: Closing-Section Compulsion

### H1. Mandatory "Summary" section

Every report ends with `## 总结 / ## Summary / ## Key Takeaways` that just
restates what the body said. Sometimes adds 3 bullets that are syntactic
copies of body sentences.

### H2. "Recommendation" or "Next Steps" section the user never asked for

The agent appends `## 建议 / ## Next Steps` even when the user only asked a
factual question.

### H3. "Q&A" or "FAQ" section

Especially in agent README output, a fake FAQ section appears with
agent-invented questions.

### H4. "Conclusion" / "结语"

Like a school essay, the agent always provides a wrap-up paragraph or section,
regardless of whether one is appropriate.

## Tell Group I: Meta-Narration in Output

### I1. The agent narrates its own work in the document

```markdown
## 分析过程
我们将通过以下步骤来分析这个问题:首先 X, 然后 Y, 最后 Z.

接下来,我们开始第一步...
```

The "接下来,我们开始第一步" sentence is meta-narration. A human writing the
same document would just do step one without announcing it.

### I2. Self-references in third person

`本文将详细介绍...` / `本报告旨在...` — these are common in formal Chinese
academic style, but the agent uses them in casual contexts where a human
would just start writing.

### I3. The agent praises its own structure

`为了让大家更清楚地理解,我们将分为三个部分讨论` — explicit meta-commentary
on the document's own organization.

## Quick Visual Diagnostic

Open the document. Without reading the words, just look at the visual pattern:

- Are most lines short (bullets) rather than wrapped paragraphs?
- Is there a `##` every screen?
- Is bold density visually high?
- Are there emoji at the start of lines?
- Are there code fences containing non-code?
- Are there tables every other section?
- Does it end with `## 总结` / `## Summary`?

If 4+ yes → high probability AI agent output, even before reading content.

## How to Humanize Markdown Agent Output

Treat formatting as a separate edit pass from content rewriting.

### Format pass

1. **Cut 50% of `##` headers.** Most are unnecessary topic boundaries.
2. **Cut 70% of `**bold**`.** Keep bold only for genuinely defining terms.
3. **Convert vertical bullet lists back to prose** where items are not strongly parallel.
4. **Remove all decorative emoji.** Keep emoji only when they carry meaning (e.g. ✅ for actually-completed items in a real tracker).
5. **Replace tables with prose** for anything that has only 2-3 rows or non-parallel content.
6. **Delete fake checklists, fake FAQs, fake "next steps".**
7. **Cut the closing "Summary" section** if it just restates the body.

### Content pass

After the format pass, the underlying prose may still be AI-flavored. Run the
checklist in `ai-tells-checklist.md` and apply rewrites.
