# Rubrics And Gates

Use these gates to decide whether a course-paper stage can advance. A gate passes when all must-pass checks are satisfied or the user explicitly accepts the remaining risk.

## Course Requirements Gate

Must pass:

- assignment type and target course are known
- required length, language, citation style, source requirements, deadline, and format are recorded
- grading rubric or likely scoring dimensions are identified
- required readings, theories, cases, or concepts are listed when provided
- academic-integrity and AI-use rules are recorded when known

Fail if:

- the draft would be written without knowing the task type or required format
- source requirements are unknown and likely to affect the paper
- the user has not provided enough assignment context and assumptions would be risky

## Topic Gate

Must pass:

- topic fits the assignment and course scope
- guiding question can be stated in one sentence
- a plausible evidence path exists
- scope is narrow enough for the word/page limit
- user approves the topic

Fail if:

- topic is a textbook survey with no arguable angle
- topic needs sources or expertise unavailable within the deadline
- topic violates the prompt or ignores required readings/theories

## Thesis Gate

Must pass:

- thesis is debatable, specific, and bounded
- thesis answers the guiding question
- "so what" significance is clear for the course
- at least one tension, counterargument, or interpretive problem is visible

Fail if:

- thesis only announces the topic
- thesis is too broad to defend in the assignment length
- thesis makes claims that sources cannot support

## Evidence Gate

Must pass:

- required course readings are considered first
- sources are inspected enough to know what they support
- major claims have evidence anchors
- quotations, paraphrases, case facts, and statistics have citation plans
- counterevidence or alternative interpretations are considered when relevant

Fail if:

- citations are invented or copied from search results without inspection
- paper relies mostly on weak web sources when scholarly sources are required
- evidence is summarized but not analyzed

## Outline Gate

Must pass:

- required structure is followed, or a justified structure is chosen when open-ended
- each section has a job
- each body section connects to the thesis
- evidence is distributed across the argument rather than dumped in one section
- conclusion has a planned "so what"

Fail if:

- outline is a list of topics rather than an argument
- sections repeat each other
- counterargument or limitation is missing when expected by the assignment

## Humanization Integrity Gate

Must pass:

- AI-flavor diagnosis or style review was run, or explicitly skipped for a rough draft
- revised prose sounds natural for the user's academic level and assignment type
- thesis, facts, citations, quotations, examples, and stance are preserved
- required AI-use disclosure remains visible when required
- changes do not make claims stronger than the evidence allows

Fail if:

- rewrite hides AI-use disclosure
- rewrite changes meaning, evidence, citation placement, or author stance without checking
- text only receives synonym swaps while retaining template structure

## Final Course Paper Gate

Must pass:

- assignment requirements and rubric expectations are satisfied
- final paper answers one clear question
- thesis, section claims, and conclusion align
- major claims have citations or clear evidence anchors
- source minimums, citation style, and reference list are checked
- word/page count and formatting are within requirements
- no unresolved high-severity review issue remains without user acceptance

Fail if:

- paper is off-topic or does not answer the prompt
- citation/reference list is inconsistent
- required source type or course reading is missing
- final artifact was not checked in the requested format when formatting matters
