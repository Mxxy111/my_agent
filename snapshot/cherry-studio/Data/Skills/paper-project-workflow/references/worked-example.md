# Worked Example

This is a compact pattern, not a fixed topic.

## User Input

The user provides a course prompt: write a 2,000-word biomedical paper with at least 15 references, recent papers, one original figure, and a required structure.

## Workflow Trace

1. Intake asks about target grade vs publication ambition, accepted language, deadline, whether a narrow experimental-design style is acceptable, and whether AI disclosure is required.
2. Requirements matrix records word count, required section order, reference minimum, recent-reference rule, figure requirement, and scoring dimensions.
3. Candidate topics are generated from broad field to narrower mechanisms. Each is scored for rubric fit, evidence, novelty, and feasibility.
4. Topic reviewers reject broad textbook topics and single-paper summaries. The coordinator recommends one focused, evidence-supported topic.
5. User approves the topic.
6. Evidence plan builds a literature matrix with primary studies, reviews, controversy sources, and methods support.
7. Structure plan maps background, design/results or argument sections, discussion, figure, and references.
8. Draft is written in full prose.
9. Review loop flags overclaiming, weak citations, missing counterevidence, and figure-caption issues.
10. Revisions tighten the claim, verify references, add disclosure, and produce DOCX/PDF or Markdown as requested.
11. Final package includes manuscript, figure, reference list, review summary, and final checklist.

## Key Pattern

The important feature is not the topic. The important feature is that topic approval, evidence sufficiency, analysis plan approval when needed, and final readiness each have a gate and a review loop.
